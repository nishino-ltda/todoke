# Plano: Multirole, "Acessar como Cliente", Auto-activation e Admin Visibility

## Premissa Fundamental

**Todo mundo é um cliente.** A role `customer` não é opcional — é a base. As roles `courier` e `partner` são **adicionais** a essa base. Um usuário SEMPRE pode navegar como cliente; os links para dashboard de courier/partner aparecem condicionalmente se ele tiver essas roles.

## Estrutura de Roles

- `admin` / `support` — roles administrativas, NÃO são clientes
- `customer` — role base universal. TODO COURIER E TODO PARTNER TAMBÉM SÃO CUSTOMER
- `courier` — role adicional (precisa de license, vehicle)
- `partner` — role adicional (precisa de business_name, etc)

---

## Problemas Identificados

1. **Sem multirole** — um courier não pode fazer pedidos como cliente; partner não pode ver o sistema como cliente
2. **Seeder não reflete multirrole** — seeders criam usuários single-role
3. **Barreira de entrada alta** — courier/partner ficam `pending` precisando de aprovação manual
4. **Admin não vê "novatos" facilmente** — sem destaque no dashboard
5. **Customer/Profile.vue é stub vazio** — sem edição real
6. **Admin/Partner/Courier não têm página de perfil** — apenas customer tem `/customer/profile`
7. **Layouts não têm link "Acessar como Cliente"** — admin, partner, courier ficam presos na própria role

## Plano de Implementação

### Fase 1: Backend — Multirole

#### 1.1 Migration: `create_role_user_table`

```php
Schema::create('role_user', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->string('role'); // 'customer', 'courier', 'partner'
    $table->timestamps();
    $table->unique(['user_id', 'role']);
});
```

#### 1.2 Model `RoleUser`

```php
class RoleUser extends Model
{
    protected $table = 'role_user';
    protected $fillable = ['user_id', 'role'];
}
```

#### 1.3 User Model — métodos `roles()`, `hasRole()`, `allRoles()`

```php
public function roleRecords(): HasMany
{
    return $this->hasMany(RoleUser::class);
}

public function hasRole(string $role): bool
{
    return $this->type === $role || $this->roleRecords()->where('role', $role)->exists();
}

public function allRoles(): array
{
    $roles = [$this->type];
    foreach ($this->roleRecords as $record) {
        $roles[] = $record->role;
    }
    return array_unique($roles);
}

public function addRole(string $role): void
{
    if ($this->type !== $role) {
        $this->roleRecords()->firstOrCreate(['role' => $role]);
    }
}
```

**Importante:** `allRoles()` SEMPRE inclui `customer` se o usuário for do tipo `courier` ou `partner`. Na prática, quando um courier se registra, ele ganha automaticamente `type='courier'` + `role_user: customer`.

#### 1.5 EnsureUserType Middleware — verificar também role_user

```php
public function handle(Request $request, Closure $next, ...$types)
{
    $user = $request->user();
    foreach ($types as $type) {
        if ($user->type === $type || $user->hasRole($type)) {
            return $next($request);
        }
    }
    return redirect()->route('login');
}
```

#### 1.7 Endpoint `POST /api/v1/users/me/roles` — Adicionar nova role

Em `API/UserController` ou novo `ProfileRoleController`:

```php
public function addRole(Request $request)
{
    $request->validate([
        'role' => 'required|in:courier,partner',
        // conditional fields for courier
        'license_number' => 'required_if:role,courier',
        'vehicle_type' => 'required_if:role,courier|in:motorcycle,car,bicycle',
        // conditional fields for partner
        'business_name' => 'required_if:role,partner',
        'business_type' => 'required_if:role,partner|in:restaurant,market,pharmacy',
        'tax_id' => 'required_if:role,partner',
        'address' => 'required_if:role,partner',
    ]);

    $user = $request->user();

    if ($request->role === 'courier' && $user->hasRole('courier')) {
        return response()->json(['message' => 'You already have this role'], 422);
    }
    if ($request->role === 'partner' && $user->hasRole('partner')) {
        return response()->json(['message' => 'You already have this role'], 422);
    }

    $user->addRole($request->role);

    // Store role-specific data
    if ($request->role === 'courier') {
        $user->update([
            'license_number' => $request->license_number,
            'vehicle_type' => $request->vehicle_type,
        ]);
    }
    if ($request->role === 'partner') {
        $user->update([
            'business_name' => $request->business_name,
            'business_type' => $request->business_type,
            'tax_id' => $request->tax_id,
            'address' => $request->address,
        ]);
    }

    return response()->json([
        'message' => 'Role added successfully',
        'user' => $user->fresh()->load('roleRecords'),
    ]);
}
```
