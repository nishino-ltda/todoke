# Plano: Unificar padrões visuais entre as telas de Checkout (Customer), Aceitação de Pedido (Partner) e Aceitação de Delivery (Courier)

## Problema

As três telas de aceitação/fluxo principal do sistema — `Customer/Checkout`, `Partner/Dashboard` (e `Partner/Orders/Index`), `Courier/Dashboard` — foram construídas em momentos diferentes e apresentam **inconsistências visuais e de padrões de código**. O objetivo é alinhá-las para uma experiência de usuário coesa e facilitar a manutenção futura.

## Análise Comparativa

### 1. Glass Card (card com efeito vidro)

| Tela | Classe | `background` | `border-radius` | `box-shadow` |
|------|--------|-------------|----------------|--------------|
| **Customer Checkout** | `.glass-card` | `rgba(255,255,255,0.8)` | `rounded="xl"` (~16px) | Vuetify `elevation="2"` |
| **Courier Dashboard** | `.glass-card` | `rgba(255,255,255,0.7)` | `24px` (CSS direto) | `0 8px 32px rgba(0,0,0,0.05)` |
| **Partner Dashboard** | N/A (usa `border elevation="0" rounded-xl`) | — | `rounded-xl` | Nenhum |

### 2. Animações de entrada

| Tela | Animação | Duração |
|------|----------|---------|
| Customer Checkout | **Nenhuma** | — |
| Partner Dashboard | `fadeIn` (transform + opacity) | 0.5s |
| Courier Dashboard | `fadeIn` (transform + opacity) | 0.6s |

### 3. Formatação de moeda

| Tela | Implementação |
|------|--------------|
| Customer Checkout | `R$ ${num.toFixed(2)}` (hardcoded) |
| Partner Dashboard | `$` ou `t('common.currency_symbol', ...)` (i18n, mas parece incompleto) |
| Courier Dashboard | `Intl.NumberFormat(locale, { style: 'currency', currency: 'BRL' })` (correto) |

### 4. Botões de ação principal

| Tela | Estilo do botão principal |
|------|--------------------------|
| Customer Checkout | `v-btn color="primary" size="large"` (padrão Vuetify) |
| Partner Dashboard | Ícone `mdi-check` `variant="text" color="success"` (apenas ícone) |
| Courier Dashboard | `block color="primary" size="x-large" font-weight-black text-none rounded-xl` + `elevation="4"` |

### 5. Estados vazios

| Tela | Implementação |
|------|--------------|
| Customer Checkout | Ícone + título + mensagem + CTA (implementado inline) |
| Courier Dashboard | Ícone + mensagem (inline, com `glass-card`) |
| Partner Dashboard | DataTable vazio (herdado do componente, sem estado vazio customizado) |

### 6. Loading states

| Tela | Implementação |
|------|--------------|
| Customer Checkout | `:loading="isSubmitting"` no botão |
| Partner Dashboard | `v-progress-circular` em charts + DataTable |
| Courier Dashboard | `v-progress-circular` no título + `:loading` no botão |

### 7. Layout de colunas

| Tela | Grid |
|------|------|
| Customer Checkout | `v-row` → `cols=12 md=7` (form) + `cols=12 md=5` (summary) |
| Partner Dashboard | Múltiplas rows (metrics 4-col, charts 8/4, table full) |
| Courier Dashboard | `v-row` → `cols=12 md=7` (active) + `cols=12 md=5` (available), or `md=12` |

## Propostas de Ajustes por Prioridade

### P1 — Glass Card padronizado

Criar um padrão único de glass card (o do Courier é o mais refinado) e aplicá-lo em todas as telas:

- Extrair classe `.glass-card` unificada para um arquivo compartilhado (ex: `resources/css/app.css`) ou manter como mixin
- Padrão proposto:
  ```css
  .glass-card {
    background: rgba(255, 255, 255, 0.7) !important;
    backdrop-filter: blur(12px);
    border-radius: 24px;
    border: 1px solid rgba(255, 255, 255, 0.3);
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.05) !important;
  }
  ```
- **Arquivos afetados:**
  - `resources/js/Pages/Customer/Checkout.vue` — atualizar `.glass-card`
  - `resources/js/Pages/Courier/Dashboard.vue` — `.glass-card` já tem estilo similar, ajustar se necessário
  - `resources/js/Pages/Partner/Dashboard.vue` — adicionar `.glass-card` aos cards de chart e tabela (remover `border elevation="0"`)
  - `resources/js/Pages/Partner/Orders/Index.vue` — adicionar `.glass-card` ao DataTable wrapper

### P2 — Animação fadeIn consistente

Adicionar `fadeIn` no Customer Checkout e padronizar duração para 0.5s em todas as telas:

- **Arquivos afetados:**
  - `resources/js/Pages/Customer/Checkout.vue` — adicionar animação `fadeIn 0.5s ease-out` no container principal
  - `resources/js/Pages/Courier/Dashboard.vue` — alterar `0.6s` → `0.5s` para consistência
  - `resources/js/Pages/Partner/Dashboard.vue` — manter `0.5s` (já está)
  - Extrair `@keyframes fadeIn` para CSS global compartilhado

### P3 — Formatação de moeda unificada

Criar um helper/composable compartilhado `useCurrency` ou `formatCurrency` e usar em todas as telas:

- **Arquivos afetados:**
  - Criar `resources/js/Composables/useCurrency.js`
  - `resources/js/Pages/Customer/Checkout.vue` — substituir `formatPrice` local por `useCurrency`
  - `resources/js/Pages/Partner/Dashboard.vue` — onde usa `$` hardcoded, usar `useCurrency`
  - `resources/js/Pages/Courier/Dashboard.vue` — já usa `Intl.NumberFormat`, migrar para `useCurrency`

### P4 — Estados vazios padronizados

Extrair um componente `EmptyState` e usar nas três telas (ou padronizar inline):
- Props: `icon`, `title`, `message`, `actionLabel`, `@action`
- **Arquivos afetados:**
  - Criar `resources/js/Components/EmptyState.vue`
  - `resources/js/Pages/Customer/Checkout.vue` — substituir estado não autenticado e carrinho vazio
  - `resources/js/Pages/Courier/Dashboard.vue` — substituir empty state de deliveries
  - `resources/js/Pages/Partner/Orders/Index.vue` — usar no DataTable vazio

### P5 — Padronizar botões de ação principal

Definir um padrão para o botão de ação primária de "aceitar" em cada papel:

- **Partner (aceitar pedido):** `mdi-check` icon button (actual icon-only) → considerar mudar para um botão com texto "Aceitar" similar ao Courier, ou manter ícone mas com tooltip
- **Courier (aceitar delivery):** manter o estilo atual (texto completo `flex-grow-1 font-weight-bold text-none rounded-lg`) que é o mais usável
- **Customer (finalizar pedido):** manter `size="large"`, adicionar `rounded-xl` e `font-weight-bold text-none` para consistência
- **Arquivos afetados:**
  - `resources/js/Pages/Customer/Checkout.vue` — atualizar estilo do botão submit
  - `resources/js/Pages/Partner/Orders/Index.vue` — se optar por mudar para botão com texto

### P6 — Loading states consistentes

- **Arquivos afetados:**
  - `resources/js/Pages/Customer/Checkout.vue` — adicionar `v-progress-circular` no fullscreen enquanto carrega dados iniciais, não só no botão
  - Padronizar uso de `v-progress-circular` com `indeterminate color="primary"`

### P7 — i18n para textos na tela de Checkout

- Verificar se todos os textos em `resources/js/Pages/Customer/Checkout.vue` e `CheckoutForm.vue` usam `$t(...)` corretamente
- Adicionar entradas faltantes nos arquivos de tradução (`pt-BR.json`, `en.json`)

## Riscos e Observações

1. **Backward compatibility:** As mudanças são puramente visuais (CSS e componentes). Nenhuma lógica de negócio é alterada.
2. **Testes:** Os seletores `data-cy` existentes não são afetados. Testes E2E (Cypress) e unitários (Vitest) não devem quebrar.
3. **Checkout já usa glass-card** com leve variação. A unificação requer atenção para não regredir o visual atual.
4. **Partner Dashboard** tem um estilo mais "plano" (`border elevation="0"`) que intencionalmente evita glass. Avaliar se partner deve migrar para glass ou manter o estilo plano como variação de tema.
