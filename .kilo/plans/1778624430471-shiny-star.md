# Plano: Implementar Suporte Completo a Addons

## Problemas Identificados

1. **Partner não consegue gerenciar addons** — A rota `/partner/addons` existe mas NÃO há link no menu lateral do parceiro (`PartnerLayout.vue`), impossibilitando a navegação.
2. **API pública de produtos não retorna addons** — `ProductController@index()` não faz eager-load de `addons`, portanto `ProductDetailsModal.vue` recebe `product.addons = undefined` e o bloco de seleção de addons não é renderizado. Afeta tanto clientes logados quanto visitantes anônimos.
3. **Seeder cria addons mas nunca associa aos produtos via pivot** — O seeder atual cria registros na tabela `addons` (só para Pizza/Burger) mas **nunca insere na pivot `product_addon`**. Mesmo que a API retornasse addons, eles estariam vazios para cada produto. Esse é o bug principal.
4. **Addons no seeder são limitados** — Só Pizza e Burger têm addons. Os demais restaurantes (Açaí, Japonês, Brasileiro) ficam sem.

## O que já funciona (não precisa mexer)

- Modelos (`Addon`, `Product`, `OrderItem`) com relacionamentos corretos
- Pivot table `product_addon` — associação produto-addon já existe no banco
- `OrderController@store` valida addon pertence ao produto via `$product->addons()->where('addon_id', ...)->exists()`
- CRUD de addons via API (`AddonController@store/update/destroy`)
- Tela `Partner/Addons/Index.vue` (CRUD de addons)
- `ProductForm.vue` com multi-select de addons
- `ProductDetailsModal.vue` com checkboxes de addons (só não renderiza porque `product.addons` é undefined)
- Carrinho (`cart.js`) persiste em localStorage com `selectedAddons` — funciona anonimamente
- `Checkout.vue` envia `addons: [{addon_id, quantity}]` no payload

## Passos da Implementação

### Passo 1: Adicionar "Addons" no menu lateral do Partner

**Arquivo:** `resources/js/Layouts/PartnerLayout.vue`

Adicionar um item no array `navItems` entre `products` e `settings`:
```js
{ titleKey: 'partner.nav.addons', icon: 'mdi-plus-box-outline', route: '/partner/addons' }
```

### Passo 2: Incluir addons na API pública de produtos

**Arquivo:** `app/Http/Controllers/API/ProductController.php`

Método `index()`:
- Adicionar `->with('addons')` na query chain
- Incluir `'addons'` no map de retorno, mesmo formato usado em `formatProduct()`:

```php
'addons' => $product->addons->map(function ($addon) {
    return [
        'id' => $addon->id,
        'name' => $addon->name,
        'price' => (float) $addon->price,
    ];
}),
```

### Passo 3: Refatorar seeder — addons por categoria + associação via pivot

**Arquivo:** `database/seeders/PartnerProductSeeder.php`

Estratégia: addons são definidos **por categoria de restaurante** e associados **aos produtos certos** via pivot `product_addon`.

Cada categoria terá seus próprios addons (ex: pizza pode ter borda recheada, bacon extra; açaí pode ter leite ninho, granola). Mas dentro do mesmo parceiro, **produtos específicos recebem addons específicos** — ou seja, uma pizza de calabresa pode ter "bacon extra" como addon, mas um guaraná (se existisse) não teria.

Estrutura:
```php
$categoryAddons = [
    'Pizza' => [
        'addons' => [
            ['name' => 'Borda Recheada', 'price' => 6.00],
            ['name' => 'Bacon Extra', 'price' => 5.00],
            ['name' => 'Queijo Extra', 'price' => 4.00],
            ['name' => 'Cheddar Extra', 'price' => 5.00],
            ['name' => 'Catupiry Extra', 'price' => 5.00],
        ],
        // Quais addons vão em quais produtos (por índice do produto)
        'product_addon_map' => [
            0 => [0, 1, 2], // Pizza Calabresa: borda, bacon, queijo
            1 => [0, 2, 4], // Pizza Margherita: borda, queijo, catupiry
            2 => [0, 1, 3], // Pizza Portuguesa: borda, bacon, cheddar
        ],
    ],
    'Burger' => [
        'addons' => [
            ['name' => 'Bacon Extra', 'price' => 4.00],
            ['name' => 'Queijo Extra', 'price' => 3.00],
            ['name' => 'Ovo', 'price' => 2.50],
            ['name' => 'Onion Rings', 'price' => 5.00],
            ['name' => 'Maionese da Casa', 'price' => 2.00],
        ],
        'product_addon_map' => [
            0 => [0, 1, 2, 4], // X-Salada: bacon, queijo, ovo, maionese
            1 => [0, 1, 3],    // X-Bacon: bacon, queijo, onion rings
            2 => [0, 1, 3, 4], // Burguer Artesanal: bacon, queijo, onion, maionese
        ],
    ],
    'Açaí' => [
        'addons' => [
            ['name' => 'Leite Ninho', 'price' => 3.00],
            ['name' => 'Granola', 'price' => 2.00],
            ['name' => 'Morango Extra', 'price' => 4.00],
            ['name' => 'Nutella', 'price' => 5.00],
            ['name' => 'Leite Condensado', 'price' => 2.50],
        ],
        'product_addon_map' => [
            0 => [0, 1, 2, 3, 4], // Açaí 500ml: todos disponíveis
            1 => [0, 1, 2, 3, 4], // Copo da Felicidade: todos
        ],
    ],
    'Japonesa' => [
        'addons' => [
            ['name' => 'Cream Cheese Extra', 'price' => 3.50],
            ['name' => 'Salmão Extra', 'price' => 8.00],
            ['name' => 'Tarê Extra', 'price' => 2.00],
            ['name' => 'Gergelim', 'price' => 1.50],
        ],
        'product_addon_map' => [
            0 => [0, 1, 3], // Combo 20 peças: cream cheese, salmão, gergelim
            1 => [0, 1],    // Temaki Salmão: cream cheese, salmão
            2 => [0, 2, 3], // Hot Roll: cream cheese, tarê, gergelim
        ],
    ],
    'Brasileira' => [
        'addons' => [
            ['name' => 'Ovo', 'price' => 2.00],
            ['name' => 'Bacon Extra', 'price' => 4.00],
            ['name' => 'Queijo Extra', 'price' => 3.00],
            ['name' => 'Banana Frita', 'price' => 3.50],
        ],
        'product_addon_map' => [
            0 => [0, 1, 2],    // Feijoada: ovo, bacon, queijo
            1 => [0, 1, 2, 3], // Marmitex: ovo, bacon, queijo, banana
        ],
    ],
];
```

Fluxo no seeder:
1. Criar/atualizar os addons do parceiro (via `Addon::updateOrCreate`)
2. Para cada produto do parceiro, verificar no `product_addon_map` quais addons associar
3. Chamar `$product->addons()->sync($addonIds)` para popular a pivot
